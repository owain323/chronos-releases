# CHRONOS 机器人架构设计

## 定位

CHRONOS 机器人在企业微信/钉钉中提供项目管理能力。**混合架构**：快速命令走确定性解析，复杂理解走 AI Agent。

## 分层模型

```
┌─────────────────────────────────────────────────────┐
│  Level 3 — AI Agent（慢路线，理解性命令）             │
│  条件：用户自然语言输入，非标准命令格式               │
│  LLM → 意图识别 → 工具调用 → 整合回复                 │
│  例："张三最近在做什么？" → LLM → getUserTasks → 分析  │
├─────────────────────────────────────────────────────┤
│  Level 1/2 — 命令解析器（快路线，确定性命令）          │
│  条件：匹配 21 个标准命令                             │
│  Regex → 提取参数 → 执行函数 → 格式化回复              │
│  例：/创建 Bug high 明天 → 正则 → handleCreate → 回复  │
├─────────────────────────────────────────────────────┤
│  数据层                                              │
│  SQLite + Drizzle ORM + 权限上下文                    │
│  botUserContext: 每人独立上下文                       │
│  botAuthCodes: 一次性绑定验证码                       │
└─────────────────────────────────────────────────────┘
```

## 路由规则

| 输入类型             | 路由                       |  延迟   |
| -------------------- | -------------------------- | :-----: |
| 匹配标准命令 (21 条) | Level 1/2 直接执行         | < 50ms  |
| 验证码 login         | callback 拦截 → Level 1    | < 30ms  |
| 注册/登录            | Level 1/2 → bcrypt         | < 200ms |
| 自然语言             | Level 3 AI Agent（待实现） |  1-3s   |

## 21 条标准命令

### 账号

| 命令                | 功能                       |
| ------------------- | -------------------------- |
| `/注册 用户名 密码` | bcrypt → 创建 CHRONOS 账号 |
| `/登录 用户名 密码` | bcrypt → 绑定机器人        |
| `login <6位码>`     | 验证码绑定                 |

## 上下文模型

每个回复统一携带上下文头：

```
▸ 项目：网站重构  |  当前：张三

📋 待办任务
...
— —
可操作：/创建 <标题> · /完成 #编号 · /我的
```

这样用户在企微里始终知道自己在哪个项目、以谁的身份操作。

## AI Agent 实现路线（待开发）

```
企微消息 → 路由判断
              │
    ┌─────────┼─────────┐
    │ 匹配命令  │ 自然语言 │
    ↓         │         ↓
  Level 1/2   │    Level 3
  直接执行     │    LLM Agent
    │         │         │
    └─────────┼─────────┘
              ↓
          回复用户
```

### Level 3 工具清单（设计草案）

| 工具名称            | 功能       | 权限             |
| ------------------- | ---------- | ---------------- |
| `getUserTasks`      | 查某人任务 | 只能查同项目     |
| `searchAll`         | 全局搜索   | 限于用户可见项目 |
| `getProjectSummary` | 项目摘要   | 项目成员         |
| `createTask`        | 创建任务   | 项目成员         |
| `getCostSummary`    | 成本总览   | 项目成员         |
| `getTeamWorkload`   | 团队负载   | 管理员           |

每个工具注入 ctx（当前用户 + 项目），在函数内部做权限校验。
