## 任务 1：修复 Analytics 路由断裂（高优先级）

**问题：** `client/src/components/AnalyticsPanel.tsx` 和 `client/src/pages/CostManagement.tsx` 调用 `trpc.analytics.getProjectStats` 和 `trpc.analytics.getCostSummary`，但后端 `server/routers.ts` 中没有定义 `analytics` 路由。运行时会报错。

**修改文件：** `server/routers.ts`

**在末尾（`}` 之前）添加：**

```typescript
  // ===== Analytics routes =====
  analytics: router({
    getProjectStats: protectedProcedure
      .input(z.object({ projectId: z.number() }))
      .query(async ({ input }) => {
        const project = await db.getProjectById(input.projectId);
        if (!project) return { totalTasks: 0, completedTasks: 0, completionRate: 0, tasksByPriority: {} };

        // Get all columns for this project
        const columns = await db.getKanbanColumnsByProjectId(input.projectId);
        const completedColumnIds = columns.filter(c => c.name === "已完成").map(c => c.id);

        // Get all tasks for each column
        let totalTasks = 0;
        let completedTasks = 0;
        const tasksByPriority: Record<string, number> = {};

        for (const col of columns) {
          const colTasks = await db.getTasksByColumnId(col.id);
          totalTasks += colTasks.length;
          if (completedColumnIds.includes(col.id)) {
            completedTasks += colTasks.length;
          }
          // Count by priority
          for (const t of colTasks) {
            const p = t.priority || "medium";
            tasksByPriority[p] = (tasksByPriority[p] || 0) + 1;
          }
        }

        return {
          totalTasks,
          completedTasks,
          completionRate: totalTasks > 0 ? Math.round((completedTasks / totalTasks) * 100) : 0,
          tasksByPriority,
        };
      }),

    getCostSummary: protectedProcedure
      .input(z.object({ projectId: z.number() }))
      .query(async ({ input }) => {
        const costs = await db.getCostEntriesByProjectId(input.projectId);
        const total = costs.reduce((sum, cost) => sum + cost.amount, 0);
        const byCategory: Record<string, number> = {};
        costs.forEach(c => {
          byCategory[c.category] = (byCategory[c.category] || 0) + c.amount;
        });
        return { total, count: costs.length, byCategory };
      }),
  }),
```

**验证方法：** 启动服务，访问任意项目页面，确认分析面板加载不报错。

## 任务 2：Dashboard 统计只取第一个项目（中优先级）

**问题：** `server/db.ts` 中的 `getDashboardStats` 函数只查询 `projectIds[0]`（即第一个项目），会统计错误的成员和任务数。

**修改文件：** `server/db.ts`，找到 `getDashboardStats` 函数（用 `// ===== Dashboard Stats =====` 注释定位）

**将整个函数替换为：**

```typescript
export async function getDashboardStats(userId: number) {
  const userProjects = db
    .select()
    .from(projects)
    .where(eq(projects.ownerId, userId))
    .all();
  const projectIds = userProjects.map(p => p.id);

  // 统计所有项目中的非重复成员数
  let totalMembers = 0;
  const allUserIds = new Set<number>();
  for (const pid of projectIds) {
    const members = db
      .select()
      .from(projectMembers)
      .where(eq(projectMembers.projectId, pid))
      .all();
    members.forEach(m => allUserIds.add(m.userId));
  }
  totalMembers = allUserIds.size;

  // 统计所有项目中不在"已完成"列的待处理任务数
  let pendingTasks = 0;
  for (const pid of projectIds) {
    const completedColumns = db
      .select()
      .from(kanbanColumns)
      .where(
        and(eq(kanbanColumns.projectId, pid), eq(kanbanColumns.name, "已完成"))
      )
      .all();
    const completedColumnIds = completedColumns.map(c => c.id);

    const allTasks = db
      .select()
      .from(tasks)
      .where(eq(tasks.projectId, pid))
      .all();
    if (completedColumnIds.length > 0) {
      pendingTasks += allTasks.filter(
        t => !completedColumnIds.includes(t.columnId)
      ).length;
    } else {
      pendingTasks += allTasks.length;
    }
  }

  return { totalProjects: userProjects.length, totalMembers, pendingTasks };
}
```

**验证方法：** 如果有多个项目，仪表板的"团队成员"和"待处理任务"数值应该比以前大（更准确了）。

## 任务 3：Bookkeeping 分录加事务保护（中优先级）

**问题：** `server/db.ts` 中 `createJournalEntry` 先插入分录，再更新余额，但**不在事务里执行**。如果余额更新失败，数据库里有分录但余额没变，账就不平了。

**修改文件：** `server/db.ts`

**找到 `createJournalEntry` 函数，将函数体改为：**

```typescript
export async function createJournalEntry(data: {
  projectId: number;
  date: string;
  description: string;
  debitAccountId: number;
  debitAmount: number;
  creditAccountId: number;
  creditAmount: number;
}) {
  // 使用 SQLite 事务确保数据一致性
  const txn = sqlite.transaction(() => {
    const now = new Date().toISOString();
    db.insert(journalEntries)
      .values({
        ...data,
        createdAt: now,
      })
      .run();

    const debitResult = db.run(
      sql`UPDATE accounts SET balance = balance + ${data.debitAmount} WHERE id = ${data.debitAccountId}`
    );
    if (debitResult.changes === 0) {
      throw new Error(`科目不存在: debitAccountId=${data.debitAccountId}`);
    }

    const creditResult = db.run(
      sql`UPDATE accounts SET balance = balance - ${data.creditAmount} WHERE id = ${data.creditAccountId}`
    );
    if (creditResult.changes === 0) {
      throw new Error(`科目不存在: creditAccountId=${data.creditAccountId}`);
    }
  });

  txn();
}
```

⚠️ **注意：** 这个函数使用了 `sqlite` 变量，需要在文件顶部导出的 `sqlite` 实例。确认文件顶部有：

```typescript
const sqlite = new Database(path.resolve(dbPath));
```

如果已经有就不用加。

**验证方法：** 在复式记账页面创建一笔分录，然后检查数据库里分录和余额都正确。

## 任务 10：Search 自动触发（低优先级）

**问题：** 全局搜索需要用户输入后再手动触发。应该加 debounce 自动搜索。

**修改文件：** `client/src/components/TopNavBar.tsx`

在文件顶部加：

```typescript
import { useCallback, useEffect, useRef, useState } from "react";
```

找到搜索关键字的 state 和查询逻辑，将 `useQuery` 的 `enabled` 改成 `keyword.length > 0`（不要求手动触发），且加一个 `useEffect` 做 300ms debounce：

```typescript
const [debouncedKeyword, setDebouncedKeyword] = useState("");

// 输入停止 300ms 后才触发搜索
useEffect(() => {
  const timer = setTimeout(() => setDebouncedKeyword(keyword), 300);
  return () => clearTimeout(timer);
}, [keyword]);

// 然后 useQuery 用 debouncedKeyword 而不是 keyword
```

## 通用规则（给所有 AI 协作者的注意事项）

### 不会改错的清单

1. ✅ **不要改 `drizzle.config.ts`** — 数据库配置不能动
2. ✅ **不要改 `package.json` 的 scripts** — `dev` / `build` 脚本保持原样
3. ✅ **不要改 `vite.config.ts`** — 除非要加依赖别名
4. ✅ **不要在 `drizzle/schema.ts` 改已有表结构** — 只能加新表，不能改已有字段
5. ✅ **不要改 `.env` 文件**
6. ✅ **不要删除或重命名 `tasknest.db`**
7. ✅ **不要改动 `server/_core/context.ts` 中的 DEFAULT_USER** — 这是本地开发认证

### 每次改完必须做的验证

```bash
# 1. TypeScript 编译检查
cd /c/Users/czj17751/WorkBuddy/2026-07-08-17-43-56/TaskNest

# 2. 服务启动无报错
npm run dev
# 观察终端输出，看到 "Server running" 才算成功

# 3. 访问前端页面
curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/

# 4. 检查 API
curl -s http://localhost:3000/api/trpc/projects.list

# 如果上面任何一步出错，撤回改动
```

### 代码风格要求

- 不要用 `any` 类型 —— 用具体的 interface 或类型
- 函数参数用解构对象（`{ param1, param2 }`）而不是多个参数
- 所有时间存 ISO 字符串（`new Date().toISOString()`）
- 页面组件用默认导出（`export default function PageName()`）
- UI 组件用命名导出
- 中文内容全部硬编码在页面里，不需要国际化
- Tailwind 类名用 `clsx` 或模板字面量拼接，不要用 `cn()` 加条件太复杂
